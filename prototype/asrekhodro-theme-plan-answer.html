۲. تبلیغات: CPT یا ACF Options؟

توصیه: CPT ad_slot — موافقم با نظر شما

Repeater در ACF Options برای ۲–۳ بنر ثابت کافی است. برای عصر خودرو نیاز دارید:

  • چند جایگاه (نوار منو ۴۳۵×۶۰، سایدبار ۱۶۰×۸۰، …)
  • فعال/غیرفعال کردن هر تبلیغ
  • تاریخ شروع/پایان (اختیاری)
  • taxonomy موقعیت: menu_strip, sidebar_left, …
  • لینک کلیک، تصویر، اولویت/ترتیب

CPT + taxonomy ad_position برای ویرایشگر راحت‌تر و مقیاس‌پذیرتر از bury کردن تبلیغات در theme options است.

پیشنهاد: CPT ad_slot + فیلدهای ACF روی هر تبلیغ (تصویر، لینک، تاریخ، active)

۴. URLهای قدیمی ایندکس‌شده گوگل — برنامه

وضعیت فعلی importer:

  • Slug: content-{contentId} (مثلاً content-263703)
  • Meta: _asrekhodro_content_id
  • Meta: _asrekhodro_old_page_url فقط اگر pageUrl در JSON باشد (نمونه ۱۰۰ پستی فعلاً pageUrl ندارد)

سایت قدیم احتمالاً مسیرهایی مثل /News/{contentId}/... (ASP.NET) دارد. با permalink پیش‌فرض وردپرس خودکار کار نمی‌کند.

برنامه SEO-safe (ریدایرکت ۳۰۱)

کاربر/گوگل → URL قدیمی → ماژول redirect تم → استخراج contentId از path → یافتن post با _asrekhodro_content_id → ۳۰۱ به permalink جدید

فاز A — قبل از launch (الزامی)

  1. ماژول Legacy Redirect در تم (یا mu-plugin کوچک): در template_redirect تطبیق الگوها:
    • /News/{id}/...
    • /Home/News/{id}/... (اگر در سایت live استفاده شود)
  2. جستجوی post: meta_key = _asrekhodro_content_id, meta_value = {id}
  3. wp_redirect( permalink, 301 )
  4. Fallback: اگر _asrekhodro_old_page_url موجود است، تطبیق path کامل

فاز B — داده (قبل از migration کامل)

  1. SQL exporter: فیلد pageUrl از DB (manifest به SingleContent اشاره دارد)
  2. re-import تا هر post متا URL قدیمی داشته باشد

فاز C — permalink جدید

  1. پست‌های جدید: slug زیبا از عنوان
  2. URLهای قدیمی ایندکس‌شده با ۳۰۱ → canonical جدید
  3. بعد از go-live: sitemap + پایش Search Console
نیاز نیست: بازتولید دقیق URL قدیمی ASP.NET به‌عنوان canonical وردپرس (سخت و شلوغ). ۳۰۱ استاندارد و درست برای گوگل است.

۵. Importer v0.2 — الان یا بعد؟

توصیه: نه v0.2 کامل — فقط v0.1.5 کوچک

همراه با تم (کوچک، پرارزش)

تغییردلیل
ذخیره overTitle / underTitle در post metaتم به آن‌ها نیاز دارد؛ در JSON هست
ذخیره contentTypeId به‌عنوان _asrekhodro_content_type_idتفکیک ویدئو/خبر بدون CPT جدا
export + import pageUrlالزامی برای redirect

حدود ۱–۲ ساعت در importer + exporter؛ تم و SEO را unblock می‌کند.

به تعویق بیفتد (این فاز نیست)

تغییردلیل تعویق
نگاشت contentTypeId → CPTهای جداشما post را برای خبر انتخاب کردید
re-slug همه پست‌ها مطابق URL قدیمredirectها SEO را پوشش می‌دهند
دانلود media به کتابخانه WPهنوز URL خارجی
پیشنهاد: Importer v0.1.5 (meta + pageUrl)، نه v0.2 کامل. تم فوراً شروع شود؛ redirect handler با تم بیاید؛ به‌روزرسانی importer هنگام export کامل SQL.
