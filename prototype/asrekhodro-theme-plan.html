هدف

ساخت تم وردپرس در مسیر asrekhodro-theme که:

  • ظاهر و ساختار prototype (صفحه استاتیک تأیید مدیریت) را پیاده کند
  • داده‌های واردشده توسط AsreKhodro Importer را بخواند
  • سریع باشد (هدف: TTFB + رندر صفحه اصلی زیر ~۱ ثانیه با کش)
  • قابل ویرایش با ACF Blocks و قالب‌بندی Timber (Twig)

فاز ۰ — همگام‌سازی با Importer (بحرانی)

وضعیت فعلی importer:

منبعوردپرس امروز
اخبارpost
دسته‌بندی‌هاcategory
برچسب‌هاpost_tag
نظراتcomments
متا_asrekhodro_content_id, _asrekhodro_image_url, _asrekhodro_author, …
قرارداد مشترک: تم و importer نباید نام فیلد جداگانه اختراع کنند — هر دو از یک قرارداد متا/ACF استفاده می‌کنند.

متا / فیلدهای پیشنهادی مشترک

_asrekhodro_content_id      → شناسه قدیمی (redirect، re-import)
_asrekhodro_image_url       → تصویر خارجی (lazy load)
_asrekhodro_content_type_id → نگاشت به CPT
overTitle / underTitle      → ACF یا post meta

Importer نسخه ۰.۲ (ارتقای کوچک)

  1. نگاشت contentTypeId → CPT (بر اساس asrekhodro.com)
  2. ذخیره overTitle و underTitle در meta/ACF
  3. اختیاری: redirect آدرس‌های قدیمی /News/{contentId}/...

فاز ۴ — جستجوی AJAX

  • Endpoint: admin-ajax.php یا REST ak/v1/search
  • جستجو در: news, video, review, car_price
  • فیلدها: title, excerpt, _asrekhodro_image_url
  • UI: جستجوی هدر prototype → dropdown نتایج (debounce 300ms)
  • اختیاری: لینک URLهای قدیمی از _asrekhodro_content_id

فاز ۵ — عملکرد (هدف زیر ۱ ثانیه)

تکنیکهدف
Timber Twig cacheکامپایل و کش قالب‌ها
Object cache (Redis)کش نتایج query بلوک‌ها
Transient per blockمثلاً news-list با TTL ۵ دقیقه
بدون jQuery (در صورت امکان)JS سبک‌تر
Critical CSSinline برای above-the-fold
Defer JSapp.js با defer
Lazy imagesloading="lazy" + decoding="async"
تصاویر خارجیبدون دانلود در import — URL مستقیم
نکته: زیر ۱ ثانیه به هاست، Redis و CDN تصاویر بستگی دارد. تم برای این هدف ساخته می‌شود؛ تنظیم production (OPcache، page cache) مرحله ops جداست.

فاز ۶ — Lazy Loading

تصاویر

  • لیست‌ها: loading="lazy" + ابعاد ثابت (163×109 برای news-list)
  • Hero: loading="eager" + fetchpriority="high"
  • اختیاری: Intersection Observer برای fade-in

لیست اخبار

  • اولین بار: ۲۰ آیتم (یا ۴۰ — قابل تنظیم در بلوک)
  • Load more: AJAX بار بعدی در گرید ۲ ستونه

تصمیم‌های مورد نیاز قبل از شروع

  1. CPT vs post: اخبار به‌صورت post بماند یا CPT جدید news؟
  2. ACF Pro: لایسنس ACF Pro موجود است؟ (برای blocks لازم است)
  3. تبلیغات: مدیریت با ACF options repeater یا CPT ad_slot؟
  4. لیست اخبار: ۴۰ آیتم یکجا یا ۲۰ + «بارگذاری بیشتر»؟
  5. Importer v0.2: همزمان با تم یا فقط تم در فاز اول؟
  6. URLهای قدیمی: redirect /News/{id}/... به permalink جدید؟
پس از تأیید شما و پاسخ به موارد بالا، شروع از فاز ۱ تا ۴ (زیرساخت تم + بلوک‌های صفحه اصلی + پل importer).
