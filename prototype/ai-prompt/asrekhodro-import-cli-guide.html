Prerequisites

  • SSH access to Linux server
  • WP-CLI installed (wp --info works)
  • WordPress root = folder containing wp-config.php
  • Plugin asrekhodro-importer active
  • JSON export in wp-content/asrekhodro-import/ (with manifest.json)
  • Theme active (for external media URLs)
Optional — wp-config.php 
define('ASREKHODRO_MEDIA_BASE_URL', 'https://www.asrekhodro.com');

run Full import (recommended)

Imports all phases in one CLI session: categories, posts, post categories, tags, relations, front sections, videos, reviews, magazines, ads, comments.

Go to WordPress root

cd /home/asrekhod/asrekhodro.net

Clean full import (first time)

wp asrekhodro-import run \
  --batch=2000 \
  --taxonomy-batch=100 \
  --reset \
  --reset-content

Options — run

OptionDefaultDescription
--batch=<n> 500 Posts imported per step. Max 10000. Use 1000–5000 on dedicated servers; lower if memory errors.
--taxonomy-batch=<n> 200 Post categories / tags / relations per step. Admin UI default is 25; CLI default is faster.
--import-dir=<path> wp-content/asrekhodro-import Custom JSON folder path.
--reset — Delete CLI resume file .wpcli-import.json and start from the beginning.
--reset-content — Before import, remove previously imported items for every export type that has data (fast SQL). Same as admin “Reset before import” checkboxes.
Do not use --reset when resuming. Resume uses the saved state file automatically.

posts Posts only

Imports news posts only — not categories, tags, or comments. Use run for a full migration.

wp asrekhodro-import posts --batch=5000 --reset
OptionDefaultDescription
--batch=<n>1000Posts per loop
--import-dir=<path>wp-content/asrekhodro-importJSON folder
--reset—Delete .wpcli-post-import.json and start from first chunk

Import phases (run order)

  1. Reset (optional, if --reset-content)
  2. Categories
  3. Posts
  4. Post categories
  5. Tags (+ legacy redirects file)
  6. Post relations
  7. Front homepage sections
  8. Videos · video categories · reviews · magazines · ads · comments

Example log lines:

[Categories] Categories 81–99 of 99 (0% overall)
[Posts] Posts 1–2000 of 50000 (1% overall)

Run in background (nohup)

So SSH disconnect does not stop the import:

cd /home/asrekhod/asrekhodro.net

# Recommended: disable LiteSpeed during import
wp plugin deactivate litespeed-cache

nohup wp asrekhodro-import run \
  --batch=2000 \
  --taxonomy-batch=100 \
  > ~/import.log 2>&1 &

# Watch log
tail -f ~/import.log

Append to existing log on resume:

nohup wp asrekhodro-import run --batch=2000 --taxonomy-batch=100 >> ~/import.log 2>&1 &

Stop a background import

1. Find the process

ps aux | grep asrekhodro-import
# or
pgrep -af "wp asrekhodro-import"

Note the PID (first number), e.g. 12345.

2. Stop gracefully

kill 12345

3. Force stop (if still running)

kill -9 12345

One-liner — stop all matching imports

pkill -f "asrekhodro-import run"
pkill -9 -f "asrekhodro-import run"
Important
  • Stopping does not roll back posts already imported.
  • Progress is saved in .wpcli-import.json after each step.
  • tail -f — press Ctrl+C only stops log viewing, not the import.

Resume after stop or error

cd /home/asrekhod/asrekhodro.net
wp asrekhodro-import run --batch=2000 --taxonomy-batch=100

Or in background:

nohup wp asrekhodro-import run --batch=2000 --taxonomy-batch=100 >> ~/import.log 2>&1 &

Do not pass --reset when resuming.

Monitor progress

# Post count increasing?
watch -n 10 'wp db query "SELECT COUNT(*) FROM wp_posts WHERE post_type=\"post\""'

# Resume file updating?
stat /home/asrekhod/asrekhodro.net/wp-content/asrekhodro-import/.wpcli-import.json

# Process still running?
ps aux | grep asrekhodro-import

Troubleshooting

Stuck at low % on posts phase

Posts are the slowest phase. Each batch of 2000 can take 15–45+ minutes. Overall 1% → 2% means ~2000 posts imported — often still working, not frozen.

Posts -499–0 of 50000 in log

Harmless display bug when crossing JSON chunk files (2500 posts per file). Import continues correctly.

Many Success: Purged all caches successfully lines

LiteSpeed Cache purges on every save. Deactivate during import:

wp plugin deactivate litespeed-cache
# after import:
wp plugin activate litespeed-cache

Memory errors

wp asrekhodro-import run --batch=500 --taxonomy-batch=50

Rough time (50k posts, batch 2000)

ConditionEstimate
LiteSpeed active, batch 20008–15+ hours
LiteSpeed off, batch 2000~6–10 hours
batch 500Slower overall, safer per step

Verify WP-CLI command exists

wp asrekhodro-import --help
wp help asrekhodro-import run
