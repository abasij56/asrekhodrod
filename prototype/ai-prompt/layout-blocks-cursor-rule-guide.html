چطور الگوی بلاک‌های Layout را در Cursor به‌صورت Rule بنویسیم؟

راهنمای تعریف قانون برای بلاک‌های غیر گوتنبرگ (ak-*) — query → View → partial.twig

پاسخ کوتاه: بله — بهترین کار ساخت یک Project Rule در .cursor/rules/ است، نه فقط توضیح در چت.

الگوی عمومی (برای همه بلاک‌های query)

پارامترهای چیدمان → query پست‌ها → View (مپ داده) → partial.twig
۱

Query

نوع پست، دسته، تعداد، نحوه گزینش → لیست پست

۲

View

هر بلاک فقط فیلدهای لازم UI را از پست برمی‌دارد (DTO ساده)

۳

partial.twig

همان فیلدها را رندر می‌کند — بدون منطق query و بدون Timber Post خام

چه چیزی بسازید؟

مسیر استاندارد Cursor:

asrekhodro-theme/.cursor/rules/layout-blocks.mdc

فایل‌های Rule با پسوند .mdc و frontmatter YAML هستند. Cursor آن‌ها را وقتی روی فایل‌های مرتبط کار می‌کنید (یا اگر alwaysApply: true باشد همیشه) به context اضافه می‌کند.

چه چیزی نسازید (به‌تنهایی)

روش کاربرد
User Rule (تنظیمات Cursor) برای همه پروژه‌ها — خوب برای اصول کلی، نه جزئیات این تم
Skill برای workflow چندمرحله‌ای — نه معماری ثابت
AGENTS.md نقشه کلی repo — مکمل خوب است، جایگزین rule دقیق نیست
README طولانی AI همیشه نمی‌خواند مگر rule به آن اشاره کند

پیشنهاد: یک Rule + glob دقیق

برای بلاک‌های layout (ak-*)، نه Gutenberg (cinfo-*):

---
description: الگوی بلاک‌های چیدمان (ak-*) — query → View → partial
globs: asrekhodro-theme/inc/blocks/ak-*/**
alwaysApply: false
---

اگر روی resolver هم کار می‌کنید، یک rule دوم:

globs: asrekhodro-theme/inc/blocks/{BlockDataResolver.php,layout/**}

محتوای پیشنهادی Rule

# بلاک‌های چیدمان (ak-*) — نه Gutenberg

## الگوی اجباری (source: query)

پارامترهای Layout Builder → query → View.php → partial.twig

1. **config.php** — `type: layout`, `source: query`, `defaults`, `partial`
2. **BlockDataResolver** — `resolve_{block}()`؛ fallback به HomepageData فقط front_page بدون override
3. **View.php** — فقط مپ به DTO؛ نه `\Timber\Post` در Twig
4. **partial.twig** — فقط کلیدهای View؛ بدون `post.meta()` و بدون query

## مرزها

- `ak-*` = Layout Builder
- `cinfo-*` = ACF/Gutenberg — این rule اعمال نشود

## مرجع

- ساده: `inc/blocks/ak-ticker/`
- split: `inc/blocks/ak-hero/`, `inc/blocks/ak-asrekhodro-featured/`
- Resolver: `inc/blocks/BlockDataResolver.php`

نکات عملی برای Cursor

  1. کوتاه بماند — زیر ~۵۰ خط؛ یک concern در هر rule.
  2. مثال واقعی — ak-ticker و ak-hero بهتر از توضیح abstract است.
  3. استثناها — source: static، composite، sidebar_widget.
  4. glob درست — cinfo-* را exclude کنید.
  5. دو rule جدا — یکی برای ak-*/**، یکی برای BlockDataResolver.php.

اختیاری: AGENTS.md

## Layout blocks (ak-*)
See `.cursor/rules/layout-blocks.mdc`. Pattern: query → View → partial.twig.

چطور بسازید؟

  1. پوشه: asrekhodro-theme/.cursor/rules/
  2. فایل: layout-blocks.mdc
  3. frontmatter + محتوا
  4. در Cursor: Rules را باز کنید و فعال بودن rule را بررسی کنید

برای ساخت خودکار فایل rule در پروژه، به Agent mode بروید و بگویید: «rule بلاک‌های layout را بساز».
